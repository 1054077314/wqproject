# PRD: 校园二手商品交易管理系统

## 0. 文档目的

本 PRD 面向开发团队、产品经理和测试人员，定义校园二手商品交易管理系统 v1 的功能需求和非功能需求。文档按功能模块组织，每个功能包含行为描述和编号的功能需求（FR），便于下游架构设计和 Epic/Story 拆分引用。术语统一见术语表。

## 1. 愿景

为本校学生提供一个集中、可信的二手商品交易平台。学生可以发布闲置物品、浏览和搜索商品、通过预约机制达成线下交易。管理员通过审核机制保证平台内容质量。

系统定位为信息撮合平台，不涉及线上支付和物流，交易双方通过预留联系方式线下完成。v1 聚焦核心交易链路可用，为后续迭代打下基础。

## 2. 目标用户

### 2.1 Jobs To Be Done

**学生（卖家）：**
- 我有闲置物品想出手，需要一个方便的发布渠道
- 我希望我的商品能被本校同学看到，而不是被外校或广告淹没

**学生（买家）：**
- 我想买二手物品省钱，需要按分类快速找到想要的东西
- 我希望商品经过审核，信息可信

**管理员：**
- 我需要控制平台内容质量，过滤垃圾信息
- 我需要了解平台运营数据

### 2.2 非用户（v1）

- 非本校学生（不支持多校区）
- 企业/商家用户
- 需要线上支付的用户

### 2.3 关键用户旅程

**UJ-1. 小王发布一台闲置笔记本电脑**
- **角色 + 场景：** 大四学生小王，毕业季想出手一台笔记本电脑
- **入口状态：** 已注册登录
- **路径：**
  1. 点击"发布商品"
  2. 填写标题、描述、价格，选择分类"电子产品"
  3. 上传 2 张实物照片
  4. 填写联系方式（微信号）
  5. 点击提交
- **高潮：** 系统提示"发布成功，等待管理员审核"
- **结局：** 商品状态为"待审核"，小王在个人中心可查看

**UJ-2. 小李浏览并预约一台二手自行车**
- **角色 + 场景：** 大二学生小李，想买一辆代步自行车
- **入口状态：** 已注册登录
- **路径：**
  1. 打开商品列表页，筛选分类"交通工具"
  2. 浏览列表，点击感兴趣的自行车进入详情
  3. 查看图片、描述、价格、卖家联系方式
  4. 点击"预约交易"
- **高潮：** 系统提示"预约成功，卖家将在个人中心看到你的预约"
- **结局：** 预约记录生成，小李在"我的预约"可查看，通过联系方式联系卖家

**UJ-3. 管理员审核新发布的商品**
- **角色 + 场景：** 管理员登录后台，查看待审核商品
- **入口状态：** 管理员已登录
- **路径：**
  1. 进入后台管理页面
  2. 查看待审核商品列表
  3. 查看商品详情（标题、描述、图片）
  4. 点击"通过"或"驳回"
- **高潮：** 商品状态变更为"已上架"或"已驳回"
- **结局：** 通过的商品出现在前台列表

## 3. 术语表

| 术语 | 定义 |
|------|------|
| **商品** | 学生发布的二手物品，包含标题、描述、价格、图片、分类、状态 |
| **商品状态** | 待审核 / 已上架 / 已驳回 / 已下架 / 已售出 |
| **分类** | 商品的类别标签，如"电子产品"、"书籍教材"、"交通工具" |
| **预约** | 买家对商品发起的交易意向记录，包含预约时间和联系方式 |
| **留言** | 商品详情页的公开评论，所有用户可见 |
| **收藏** | 学生对商品的标记，便于后续查找 |
| **管理员** | 由开发者手动创建的后台管理账号，拥有审核和管理权限 |

## 4. 功能需求

### 4.1 用户注册与登录

**描述：** 学生通过用户名+密码注册账号，注册后可登录获取 Token（有效期 7 天）。访客可浏览商品但不能发布、预约、留言、收藏。[ASSUMPTION: 用户名全局唯一，不支持修改]

**功能需求：**

#### FR-1: 用户注册

学生可以通过用户名和密码注册账号。

**验收条件：**
- 用户名长度 3-50 字符，全局唯一
- 密码长度 ≥ 6 字符
- 密码以 bcrypt 加密存储
- 用户名重复返回 409 错误
- 注册成功返回 201

#### FR-2: 用户登录

已注册学生可以通过用户名和密码登录，获取 Token。

**验收条件：**
- 用户名或密码错误返回 401
- 登录成功返回 Token 和用户基本信息
- Token 有效期 7 天
- Token 格式：Bearer {token}

#### FR-3: 获取当前用户信息

登录用户可以通过 Token 获取自己的基本信息。

**验收条件：**
- 有效 Token 返回用户 id 和 username
- 无效/过期 Token 返回 401
- 无 Token 返回 401

### 4.2 商品管理

**描述：** 登录学生可以发布商品，发布后进入待审核状态。管理员审核通过后商品上架，前台可见。商品最多上传 3 张图片。[ASSUMPTION: 图片存储在本地文件系统，后续可迁移至 OSS]

**功能需求：**

#### FR-4: 发布商品

登录学生可以发布商品，包含标题、描述、价格、分类、图片、联系方式。

**验收条件：**
- 必填字段：标题、描述、价格、分类、联系方式
- 图片可选，最多 3 张
- 发布后状态为"待审核"
- 价格为正数，精确到分
- 标题长度 ≤ 100 字符，描述长度 ≤ 2000 字符

#### FR-5: 编辑商品

商品发布者可以编辑自己未上架的商品。

**验收条件：**
- 只能编辑自己的商品
- 已上架商品不可编辑 [ASSUMPTION: 已上架商品需下架后重新编辑]
- 编辑后状态重置为"待审核"

#### FR-6: 删除商品

商品发布者可以删除自己的商品。

**验收条件：**
- 只能删除自己的商品
- 删除为软删除（状态变更为"已下架"）
- 已有预约的商品不可删除 [ASSUMPTION: 需先取消所有预约]

#### FR-7: 商品列表

所有用户（含访客）可以浏览已上架商品列表，支持分页和分类筛选。

**验收条件：**
- 只显示状态为"已上架"的商品
- 支持按分类筛选
- 分页参数：page（默认 1）、page_size（默认 20）
- 返回商品基本信息（标题、价格、首图、分类）
- 按发布时间倒序排列

#### FR-8: 商品详情

所有用户（含访客）可以查看商品详情。

**验收条件：**
- 返回商品全部信息（标题、描述、价格、图片列表、分类、卖家联系方式）
- 返回留言列表
- 返回是否已被当前用户收藏（登录状态）
- 返回预约数量 [ASSUMPTION: 显示预约数但不显示预约者信息]

#### FR-9: 我的发布

登录用户可以查看自己发布的商品列表。

**验收条件：**
- 显示所有状态的商品
- 支持按状态筛选
- 显示预约数量

### 4.3 商品审核

**描述：** 管理员可以审核待上架商品，通过后商品在前台可见。

**功能需求：**

#### FR-10: 审核商品

管理员可以对待审核商品执行"通过"或"驳回"操作。

**验收条件：**
- 只有管理员可以操作
- 通过：状态变更为"已上架"
- 驳回：状态变更为"已驳回"，必须填写驳回原因
- 操作记录保存

#### FR-11: 待审核列表

管理员可以查看所有待审核商品。

**验收条件：**
- 按提交时间排序
- 显示商品基本信息
- 支持分页

### 4.4 分类管理

**描述：** 商品分类由管理员维护。[ASSUMPTION: 分类为两级结构，v1 只做一级分类]

**功能需求：**

#### FR-12: 分类列表

所有用户可以获取商品分类列表。

**验收条件：**
- 返回分类 id 和名称
- 按排序字段升序

#### FR-13: 管理分类

管理员可以增删改分类。

**验收条件：**
- 只有管理员可以操作
- 删除分类时，该分类下有商品则不可删除
- 分类名称不可重复

### 4.5 预约交易

**描述：** 登录学生可以对已上架商品发起预约，生成预约记录。卖家在个人中心查看预约列表。[ASSUMPTION: 一个用户对同一商品只能预约一次]

**功能需求：**

#### FR-14: 发起预约

登录学生可以对已上架商品发起预约。

**验收条件：**
- 只能预约他人的商品
- 只能预约"已上架"状态的商品
- 同一用户对同一商品只能预约一次
- 预约成功后生成记录

#### FR-15: 我的预约（买家）

登录用户可以查看自己发起的预约列表。

**验收条件：**
- 显示预约的商品信息和预约时间
- 按预约时间倒序

#### FR-16: 我的预约（卖家）

登录用户可以查看自己商品被预约的列表。

**验收条件：**
- 显示预约者信息和预约时间
- 按预约时间倒序

### 4.6 收藏

**描述：** 登录学生可以收藏/取消收藏商品。

**功能需求：**

#### FR-17: 收藏/取消收藏

登录用户可以收藏或取消收藏已上架商品。

**验收条件：**
- 点击收藏按钮切换状态
- 只能收藏"已上架"商品
- 不能收藏自己的商品

#### FR-18: 我的收藏

登录用户可以查看自己的收藏列表。

**验收条件：**
- 显示收藏的商品基本信息
- 按收藏时间倒序
- 支持分页

### 4.7 留言

**描述：** 登录用户可以在商品详情页公开留言。

**功能需求：**

#### FR-19: 发表留言

登录用户可以在商品详情页发表留言。

**验收条件：**
- 留言内容非空，长度 ≤ 500 字符
- 访客不可留言
- 留言立即显示

#### FR-20: 查看留言

所有用户（含访客）可以查看商品的留言列表。

**验收条件：**
- 按时间正序排列
- 显示留言者用户名和留言时间

### 4.8 管理员后台

**描述：** 管理员后台提供用户管理、商品审核和数据统计功能。

**功能需求：**

#### FR-21: 用户管理

管理员可以查看用户列表，禁用/启用用户。

**验收条件：**
- 显示用户列表（用户名、注册时间、状态）
- 禁用后用户无法登录
- 管理员账号不可禁用自己

#### FR-22: 数据统计

管理员可以查看平台运营数据。

**验收条件：**
- 用户总数
- 商品总数（按状态分：待审核、已上架、已驳回、已下架、已售出）
- 今日新增商品数
- 待审核商品数

## 5. 非目标（明确排除）

- 线上支付/担保交易
- 即时通讯/私信
- 商品搜索（关键词搜索）
- 商品推荐算法
- 多图轮播/视频
- 手机号/实名认证
- 举报/投诉机制
- 多校区支持
- 商品评价/打分
- 物流/快递集成

## 6. MVP 范围

### 6.1 包含

- 用户注册/登录（FR-1, FR-2, FR-3）
- 商品 CRUD + 图片上传（FR-4, FR-5, FR-6）
- 商品列表和详情（FR-7, FR-8, FR-9）
- 商品审核（FR-10, FR-11）
- 分类管理（FR-12, FR-13）
- 预约交易（FR-14, FR-15, FR-16）
- 收藏（FR-17, FR-18）
- 留言（FR-19, FR-20）
- 管理员后台（FR-21, FR-22）

### 6.2 不包含（v1）

- 商品搜索 — v1 只做分类筛选，关键词搜索留到 v2
- 图片压缩 — v1 原图上传，后续加压缩
- 手机号绑定 — v1 只用联系方式字段，后续可加实名
- 批量审核 — v1 逐条审核，后续加批量操作

## 7. 成功指标

**主要指标：**
- **SM-1:** 核心流程可用 — 注册→发布→审核→浏览→预约全链路跑通，无阻断性 bug。验证 FR-1~FR-22。
- **SM-2:** 接口响应时间 — 列表/详情接口 < 500ms（本地环境）。验证 FR-7, FR-8。

**次要指标：**
- **SM-3:** 代码质量 — 后端测试覆盖率 ≥ 60%。
- **SM-4:** 模块清晰度 — 代码按 model → serializer → view → url 分层，无跨层调用。

**反指标（不应优化）：**
- **SM-C1:** 不追求 UI 美观度 — v1 功能优先，UI 可在后续迭代优化。
