# 产品简报：校园二手商品交易管理系统

## 概述

面向本校学生的二手商品交易平台。学生注册后可发布、浏览、收藏、留言、预约交易二手物品；管理员负责商品审核、用户管理和数据监控。系统只做信息撮合，不涉及线上支付，交易双方线下完成。

**技术栈：** Django 4.1 + MySQL 5.7 / React 19 + TypeScript + Vite 6 + Tailwind CSS 4 / 自定义 Token 认证

## 解决的问题

**现状痛点：**
- 学生毕业/换季产生大量闲置物品，缺乏统一渠道处置
- 现有方式（微信群、朋友圈、贴吧）信息分散、难以检索、无分类筛选
- 买卖双方缺乏信任机制，交易体验差

**系统价值：**
- 集中商品信息，按分类检索，提升找货效率
- 审核机制保证商品质量，减少垃圾信息
- 预约+联系方式模式降低交易门槛

## 核心功能

### 用户端（学生）

| 功能 | 说明 |
|------|------|
| 注册/登录 | 用户名+密码，登录后获取 Token（7 天有效期） |
| 商品浏览 | 访客可浏览商品列表和详情，按分类筛选 |
| 商品发布 | 登录用户发布商品，上传最多 3 张图片，发布后进入待审核状态 |
| 商品详情 | 查看商品信息、图片、卖家联系方式、公开留言 |
| 预约交易 | 买家点击预约，生成预约记录，卖家在个人中心查看 |
| 收藏 | 收藏/取消收藏商品 |
| 留言 | 在商品详情页公开留言，所有人可见 |
| 个人中心 | 查看我的发布、我的预约、我的收藏 |

### 管理端（管理员）

| 功能 | 说明 |
|------|------|
| 商品审核 | 审核待上架商品，通过/驳回 |
| 用户管理 | 查看用户列表，禁用/启用用户 |
| 数据统计 | 用户总数、商品总数（按状态分）、今日新增商品、待审核数量 |

## 用户角色

| 角色 | 说明 |
|------|------|
| 访客 | 可浏览商品列表和详情，不可发布/预约/留言/收藏 |
| 学生 | 注册登录后拥有全部用户端功能 |
| 管理员 | 由开发者手动创建账号，拥有后台管理权限 |

## 交易流程

```
学生A发布商品 → 管理员审核通过 → 商品上架
                                      ↓
学生B浏览商品 → 点击预约 → 生成预约记录
                                      ↓
学生A在个人中心看到预约 → 双方通过联系方式线下交易
```

## 范围边界

### 第一版包含

- 用户注册/登录
- 商品 CRUD + 图片上传（最多 3 张）
- 商品分类管理
- 商品审核流程（待审核 → 通过/驳回）
- 商品列表（分页、分类筛选）
- 商品详情（图片、留言）
- 预约交易
- 收藏功能
- 公开留言
- 管理员后台（审核、用户管理、数据统计）

### 第一版不包含

- 线上支付/担保交易
- 即时通讯/私信
- 搜索功能（关键词搜索）
- 商品推荐算法
- 多图轮播/视频
- 手机号/实名认证
- 举报/投诉机制
- 多校区支持

## 成功标准

| 指标 | 目标 |
|------|------|
| 核心流程可用 | 注册→发布→审核→浏览→预约 全链路跑通 |
| 接口响应 | 列表/详情接口 < 500ms |
| 代码质量 | 后端测试覆盖率 ≥ 60%，前端核心组件有测试 |
| 可维护性 | 模块清晰分离（model → serializer → view → url） |

## 风险与假设

| 风险/假设 | 应对 |
|-----------|------|
| [假设] 用户愿意留下联系方式 | 第一版不做验证，后续可加手机号绑定 |
| [假设] 管理员审核不会成为瓶颈 | 后续可加自动审核或批量审核 |
| [风险] 图片上传无压缩，可能占用存储 | 第一版接受，后续加图片压缩 |
